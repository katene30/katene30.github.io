What are the best practices associated with using classes vs. ids?

In order to understand which practises are best for each. Let’s get a better idea on what exactly classes and IDs are, and what we use them for. Classes and IDs allows programmers full control over their website.
Firstly, we should understand that an HTML file contains the structure of the site and a CSS file contains the styling of the page. In knowing that we can understand how and why we use classes and IDs.
So HTML classes and IDs are selectors or hooks that you can assign to HTML elements to make them more specific. They act as additional identifiers to assign CSS to.

For example, if I wanted to make this text look green and bigger than the rest. I have!

I did that by:

  1. Assigning the string of text I wanted to make look different an ID
  2. I assigned that specific ID the correct CSS by using the prefix #

And that’s it!
You can do the same thing with classes but instead of # as a prefix you would use a .
And to call upon that CSS you would use class="example-class" instead of id="example-id"
But the main property that differs the two selectors is that with an ID it is meant to be unique to the page whereas a class isn’t. To explain, classes can be used in multiple HTML elements on the same page. And an ID can only be used once. IDs also have additional unique abilities, such as using them as bookmarks.
IDs have a unique ability to allow you to jump to a specific element on a page.

Click here to see what I mean!

To sum that all up we know that:

  • IDs and classes are both selectors to help us customize a webpage
  • In order to specifically style one element we should use IDs
  • If there are multiple elements we want to change, we would use classes
  • IDs have different abilities including the ability to jump to different points in a page.

So that just shows how powerful it is to know and understand classes and IDs for your HTML and there is no doubt that any avid programmer should learn about, and how to use them.
